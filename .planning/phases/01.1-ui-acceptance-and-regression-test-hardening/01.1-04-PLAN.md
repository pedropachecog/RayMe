## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser-rendered UI -> acceptance claim | A route-mocked test can pass HTTP expectations while a page exception breaks the UI. |
| Untrusted character prose -> browser DOM | Malicious imported card text must remain sanitized in full-path tests. |
| Chat action route mocks -> persisted UI state | Regenerate, swipe, continue, stale flags, and reload must reflect backend-returned state, not local-only state. |

| T-01.1-04-02 | Elevation of Privilege | Imported malicious character prose | mitigate | Full-path spec preserves malicious HTML assertions and checks `img[src="x"]` is absent. |
| T-01.1-04-03 | Tampering | Chat action state | mitigate | Full-path spec asserts backend-selected alternate IDs after regenerate, swipe, continue, reload, and second send. |
