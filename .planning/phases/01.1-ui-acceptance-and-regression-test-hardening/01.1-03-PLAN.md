<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Settings form -> RayMe API | User-entered endpoint values must be persisted before server-side probes run. |
| Browser -> LLM provider | Browser code must not call the local LLM or OpenAI provider directly. |
| Server-side settings -> UI status | API keys must remain server-side and absent from status/toast/console output. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01.1-03-01 | Spoofing | Settings connection tests | mitigate | E2E asserts `PATCH /api/settings` with current form values immediately precedes each test endpoint POST. |
| T-01.1-03-02 | Information Disclosure | LLM API key handling | mitigate | E2E and pytest assert no raw key in browser text, status payloads, or test responses. |
| T-01.1-03-03 | Tampering | Local LLM endpoint config | mitigate | E2E asserts `http://192.168.1.190:8001/v1` is Settings-entered test data only and production source does not hardcode it. |
</threat_model>
