<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Playwright page -> Svelte app | Browser-visible UI can fail while HTTP routes still return 200. |
| Browser requests -> RayMe backend/API | Tests must catch direct provider calls and raw key leakage from browser code. |
| Test helper allowlists -> failures | Broad allowlists could hide bootstrap, CSP, or runtime errors. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01.1-01-01 | Denial of Service | Playwright browser checks | mitigate | `installBrowserErrorGuard` fails tests on `console.error` and `pageerror` instead of allowing blank-page/bootstrap failures to pass. |
| T-01.1-01-02 | Information Disclosure | Browser request helpers | mitigate | `expectRayMeApiRequest` rejects raw `sk-` query strings and direct provider origins including `api.openai.com` and `192.168.1.190:8001`. |
| T-01.1-01-03 | Tampering | Helper allowlist | mitigate | `allowConsoleErrors` uses explicit `RegExp[]` matchers and defaults to no suppressed console errors. |
</threat_model>
