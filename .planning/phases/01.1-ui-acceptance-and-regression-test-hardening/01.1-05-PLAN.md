<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local source tests -> deployed OMEN-PC runtime | Local green tests can be invalid if the LAN runtime is stale or broken. |
| Playwright automation -> Android physical device | Browser automation cannot prove physical Android trust-store and touch behavior. |
| Browser test -> provider/runtime endpoints | Live tests must not bypass the Web UI Settings/API path or leak keys. |
| Agent handoff -> product-owner testing | Manual testing must not be first-line QA or vague debugging work. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01.1-05-01 | Spoofing | Live LAN runtime | mitigate | Functional live spec uses explicit `RAYME_LIVE_WEB_URL=https://192.168.1.199:8443` and AI health `https://192.168.1.199:9443/health`, not localhost. |
| T-01.1-05-02 | Denial of Service | Browser readiness | mitigate | Live Playwright spec asserts Settings, portrait persistence, and full text flow with console/page-error guard before Android handoff. |
| T-01.1-05-03 | Information Disclosure | Settings status, console, screenshots | mitigate | Spec requires blank API-key path for local LLM and forbids direct provider calls or status/console leakage of keys. |
| T-01.1-05-04 | Repudiation | Product-owner checkpoint | mitigate | Validation file records green approval or exact blocker with concrete URLs, live spec name, and date. |
| T-01.1-05-05 | Tampering | Default local E2E suite | mitigate | `playwright.config.ts` excludes `**/live-*.spec.ts` unless `RAYME_ENABLE_LIVE_E2E=1`, and verification asserts default listing omits the live spec. |
</threat_model>
