<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser image element -> RayMe portrait API | Broken or stale portrait URLs can leave UI looking successful while images fail to load. |
| Imported/uploaded file -> character persistence | Untrusted card/image input crosses into stored character metadata and blobs. |
| Route mocks -> UI assertions | JSON-only mocks can hide missing actual image requests. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01.1-02-01 | Tampering | `portrait-persistence.spec.ts` import/upload routes | mitigate | Tests assert uploaded filenames, active `portrait_asset_id`, replacement asset identity, and removal `portrait_url: null`. |
| T-01.1-02-02 | Denial of Service | Browser image rendering | mitigate | Tests fulfill actual `image/png` portrait URLs and assert visible `<img>` elements across Editor, Gallery, Home, and Chat. |
| T-01.1-02-03 | Information Disclosure | Browser requests | mitigate | `expectRayMeApiRequest` prevents direct provider/key-bearing browser requests during portrait flows. |
</threat_model>
