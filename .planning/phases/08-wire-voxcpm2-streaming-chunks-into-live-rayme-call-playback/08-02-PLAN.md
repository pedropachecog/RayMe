<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TTS adapter -> CallSession | Runtime-produced chunks enter call-session event and playback lifecycle. |
| Interrupt/control route -> CallSession | User control can cancel active generation and queued playback. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-02-01 | Information Disclosure | `CallSession.speak_text` errors | mitigate | Preserve fixed `call_tts_failed` public messages and never attach tracebacks, local paths, cache paths, or raw model ids to events. |
| T-08-02-02 | Denial of Service | reference audio and VoxCPM2 options | mitigate | Build streaming requests through `TtsSynthesisInput` and `_decode_reference_audio_b64`, preserving byte limits and bounded option fields. |
| T-08-02-03 | Elevation of Privilege | production model path | mitigate | Streaming branch uses the resident adapter selected by model manager; do not add CPU runtime fallback. |
| T-08-02-04 | Tampering | interrupt after first chunk | mitigate | Check `_cancelled_ai_turns` before every chunk enqueue and call existing `stop_current()`/buffer drain through `cancel_ai_turn`. |
| T-08-02-05 | Repudiation | fallback evidence | mitigate | Include `streaming_used`, `fallback_used`, and `whole_wav_fallback_used` in `tts_playback` for whole-WAV and streamed paths. |
</threat_model>
