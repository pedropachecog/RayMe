## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser to character API | User chooses a voice ID to attach to character state. |
| Voice delete to character reads | Deleted voice references must remain explainable and non-crashing. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-11-01 | Tampering | `default_voice_id` writes | mitigate | Validate voice exists and is not deleted before assignment. |
| T-02-11-02 | Denial of Service | Character reads | mitigate | Hydrate missing/deleted voices as `unavailable`, not exceptions. |
| T-02-11-03 | Repudiation | Voice delete references | mitigate | Return readable labels and tombstone deleted name when available. |
| T-02-11-04 | Information Disclosure | Character response | accept | Voice names are local user-created display text and already visible in library. |
</threat_model>
