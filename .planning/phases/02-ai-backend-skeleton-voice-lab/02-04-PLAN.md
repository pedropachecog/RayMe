<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser upload to Web UI storage | Audio bytes and filenames become local files if accepted. |
| Voice references to character records | Deletion state must not corrupt character data. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04-01 | Tampering | `voice_assets.storage_path` | mitigate | Generate flat `{asset_id}.{extension}` names and reuse `atomic_write_blob`. |
| T-02-04-02 | Denial of Service | Upload validation | mitigate | Enforce 25 MiB limit and reject empty/unsupported files before AI processing. |
| T-02-04-03 | Information Disclosure | Blob paths | mitigate | Store relative internal blob names only; do not expose filesystem paths. |
| T-02-04-04 | Tampering | `characters.default_voice_id` | mitigate | Use stable ID references plus soft-delete state in `voices.deleted_at`. |
</threat_model>
