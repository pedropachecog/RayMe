# Phase 02 - UI Design Contract

> Visual and interaction contract for Phase 2 frontend work. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope And Sources

Phase 2 extends the existing local Svelte app with the Voice Lab, Voice Library, default-voice assignment, and expanded Settings controls needed before the first call phase. The primary user path is: Settings connection tests -> Voice Lab upload -> auto-transcript -> inline transcript edit -> select TTS engine -> synth-preview -> save voice -> Voice Library test-play/rename/delete -> assign character default voice -> Gallery shows voice state.

| Font | Manrope for display/heading where available, Inter/system sans for body/label |

Design-system contract:

- `components.json` is absent and shadcn is not initialized. Because this is SvelteKit, the shadcn React initialization gate is not applicable.
- Keep `web-ui/client/src/app.css` variables as the source of truth for tokens. Add only Phase 2 component styles that consume those variables.
- Preserve the approved True Dark / Ethereal Core treatment: tonal layering, no pure white text, no marketing hero, no fake unavailable features, no account/billing/logout controls.
- Preserve the no-line rule for layout sectioning. Use background shifts, spacing, and ghost outlines for state clarity, not 1px dividers.
- Use glass surfaces only for floating menus, modals, sticky preview controls, and confirmation dialogs: `surface-container-highest` at 60% opacity with `backdrop-filter: blur(20px)`.
- Voice Lab may use compact waveform/progress visuals for uploaded samples and synth previews. Do not introduce the full call Voice Visualizer in Phase 2.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, waveform bar gaps, dense row action spacing |
| sm | 8px | Compact element spacing, upload guidance chips, library row metadata gaps |
| md | 16px | Default element spacing, input padding, card internal padding, transcript editor padding |
| lg | 24px | Section padding, Settings group spacing, Voice Lab step spacing |
| xl | 32px | Page gutters, Voice Lab main column gap, library-to-preview gap |
| 2xl | 48px | Major vertical separation, empty-state breathing room |
| 3xl | 64px | Desktop top/bottom workspace padding only |

Exceptions: minimum interactive target is 44px for icon buttons, audio play/pause controls, upload/dropzone actions, engine selector options, rename/delete/test-play controls, Settings `Test Connection` buttons, toggles, sliders, and mobile navigation.

Layout dimensions:

- Voice Lab desktop layout uses a two-column workspace: primary creation flow minmax `520px, 760px`; Voice Library/status rail minmax `320px, 420px`. Collapse to a single column below the tablet breakpoint.
- Upload dropzone minimum height is 176px desktop and 148px mobile. Empty, drag-active, invalid-file, uploading, and uploaded states keep the same dimensions.
- Transcript editor minimum height is 160px and max height is 360px before internal scroll. Editing transcript text must not move preview/save controls off-screen.
- Voice Library rows are stable at 88px minimum height. Rename mode, test-play loading, and delete-blocked badges must not resize neighboring rows.
- Character Editor voice assignment control fits inside the existing left identity/meta column without changing portrait preview dimensions.

## Typography

Use exactly these four type sizes for product UI text.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 | 1.5 |
| Label | 12px | 600 | 1.3 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.15 |

Typography rules:

- Use only weights 400 and 600.
- Letter spacing is 0.
- Voice names, engine names, endpoint URLs, model names, and reference transcript text use Body. Long values wrap or truncate without horizontal page scroll.
- Engine caveats and duration warnings use Label plus icon, not Display.
- Display is reserved for screen-level titles only. Do not use hero-scale type inside Voice Library rows, Settings panels, or warning chips.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#060e20` | App background, Voice Lab canvas, page base |
| Secondary (30%) | `#091328`, `#141f38`, `#192540` | Sidebar, panes, cards, Settings panels, upload zone, transcript editor, Voice Library rows, menus |
| Accent (10%) | `#b6a0ff`; pulse gradient `linear-gradient(135deg, #b6a0ff 0%, #70aaff 100%)` | Specific elements listed below |
| Destructive | `#ff716c` | Destructive actions only |

Accent reserved for: primary Voice Lab CTA, active Voice Lab step, selected TTS engine, focused transcript editor, synth-preview progress, successful save emphasis, assigned default voice badge, active Settings connection-test affordance, and Save button.

Additional color rules:

- Use `#00e3fd` only for active communication/confirmation states: transcribing, synthesizing, connected endpoint, healthy AI backend, and ready audio preview. Pair it with text and icon.
- Use `#ff716c` for invalid file type, failed transcription, failed synthesis, blocked delete, destructive confirmation, and connection failure indicators. Do not use destructive fill except inside confirmation dialogs.
- Use the existing text colors: `#dee5ff` primary and `#9eaad5` secondary. Do not use `#ffffff`.
- Qwen3-TTS caveat badges use secondary surfaces with accent/cyan icon treatment, not destructive color. Qwen is opt-in and non-default, not an error.
- Sample duration warnings are non-blocking when the backend can still process the file. Use warning copy and an `AlertTriangle` icon; reserve destructive styling for unsupported or unreadable files.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Preview Voice |
| Save CTA after preview succeeds | Save Voice |
| Empty state heading | No voices yet |
| Empty state body | Upload a 6-15 second WAV, MP3, or FLAC sample to create the first voice. |
| Voice Lab error state | RayMe could not process this sample. Check the file format, keep the sample near 6-15 seconds, and try again. |
| Transcript pending state | Transcribing sample... |
| Transcript ready helper | Review and edit the transcript before previewing the voice. |
| Preview required state | Preview the voice before saving so transcript issues are caught early. |
| Qwen engine caveat | Qwen3-TTS 0.6B-Base is experimental and non-default; use it only when you want the Apache-2.0 path despite current latency and accent caveats. |
| Voice Library custom text placeholder | Type a test phrase |
| Settings endpoint error | RayMe cannot reach this endpoint. Check the URL, save Settings, and run Test Connection again. |
| Destructive confirmation | Delete voice: Delete this voice? Characters or chats using it must be reassigned first. |
| Destructive confirmation | Delete blocked voice: This voice is in use. Reassign the listed characters or chats before deleting it. |
| Destructive confirmation | Replace default voice: Replace this character's default voice? Existing chats keep their history. |
| Destructive confirmation | Clear AI audio default change: Turn off AI audio saving? Future AI call audio will not be stored for replay. |

Copy rules:

- Use direct operational labels: `Upload Sample`, `Transcribe Sample`, `Preview Voice`, `Save Voice`, `Test Voice`, `Rename Voice`, `Delete Voice`, `Assign Voice`, `Test Connection`, `Save Settings`.
- Use exact engine labels: `F5-TTS`, `XTTS v2`, `Qwen3-TTS 0.6B-Base`.
- Always describe Qwen as opt-in/non-default with caveats. Do not hide it in current Phase 2 requirements.
- Error copy must name the failing operation and the next action. Do not collapse upload, transcript, synthesis, Settings, and delete failures into generic network errors.
- Do not expose backend tracebacks, local file paths, API keys, or raw model exception text in product UI.

## Screen Contracts

### App Shell Navigation

- Add `Voice Lab` as a real top-level destination only when the route is implemented. Desktop rail and mobile bottom navigation may include Home, Gallery, Voice Lab, and Settings.
- Use a `Mic` or `FileAudio` icon for Voice Lab. Keep existing Home, Gallery, Settings ordering stable: Home -> Gallery -> Voice Lab -> Settings.
- Chat and Character Editor remain contextual destinations with back affordances.
- Do not add Call navigation or a disabled call button in Phase 2.

### Voice Lab Creation Flow

- Voice Lab is a functional workspace, not an explainer page. The first visible surface is the upload/create flow with the Voice Library visible on desktop.
- Required flow steps are visually explicit: `1 Upload`, `2 Transcript`, `3 Engine`, `4 Preview`, `5 Save`. Use compact step chips; do not turn them into a wizard that hides prior steps.
- Upload accepts only WAV, MP3, and FLAC. Unsupported files show an inline error before upload.
- Guidance copy must state the 6-15 second recommended envelope and mono preference. Clips outside the envelope show warnings:
  - Under 6 seconds: `This sample may be too short for a stable clone. Use 6-15 seconds when possible.`
  - Over 15 seconds: `This sample is longer than recommended and may take longer to process. Use 6-15 seconds when possible.`
- Auto-transcript appears inline in the same flow. The transcript is editable before preview and save.
- If transcription fails, keep the uploaded sample and allow retry or manual transcript entry if the backend contract supports it. Do not force the user to re-upload.
- Engine choice uses a segmented control or radio-card group with three options: `F5-TTS` default selected, `XTTS v2`, and `Qwen3-TTS 0.6B-Base` opt-in/non-default.
- Engine cards show concise requirements:
  - F5-TTS: transcript required, default.
  - XTTS v2: transcript stored for portability.
  - Qwen3-TTS 0.6B-Base: transcript required, experimental/non-default, latency/accent caveats.
- `Preview Voice` is disabled until there is an uploaded sample, a non-empty voice name, an editable transcript state, and a selected engine.
- `Save Voice` is disabled until synth-preview succeeds for the currently selected sample, transcript, engine, and voice name. Editing any of those fields after preview invalidates the preview and returns the primary action to `Preview Voice`.
- Preview uses a stock phrase by default and may allow editable preview text. Preview audio controls must include play/pause, retry synthesis, and a visible synthesizing state.
- Save success keeps the user in Voice Lab, adds/updates the Voice Library row, and clears the creation flow only after confirmation that the saved voice appears in the library.

### Voice Library

- Voice Library lists saved voices with name, engine, transcript presence, created/updated timestamp, and assignment status.
- Required row actions: `Test Voice`, `Rename Voice`, `Delete Voice`.
- Test-play supports custom text. Empty custom text falls back to a stock phrase.
- Test-play loading is row-scoped. One row synthesizing must not block renaming or reading other rows, but concurrent playbacks should stop or pause prior preview audio.
- Rename uses inline edit on desktop and a modal/sheet on mobile. Save and cancel controls remain 44px minimum.
- Delete requires confirmation. If the backend blocks deletion due to referents, show a blocker dialog listing each character default and chat override by readable name.
- If a safe reassignment path is implemented, the reassignment control must list target voice names and use `Reassign and Delete` as the destructive confirmation label.
- Library empty state uses the Copywriting Contract and routes focus to the upload dropzone.

### Character Editor Default Voice

- Add default voice assignment to the existing Character Editor identity/meta column below portrait and core identity fields.
- Control label: `Default voice`.
- Empty state inside the control: `No voice assigned`.
- If no voices exist, show a secondary action `Create Voice` that routes to Voice Lab without losing unsaved editor changes. If unsaved changes cannot be preserved, confirm before navigating.
- Voice options show voice name, engine, and a Qwen caveat chip when applicable.
- Assigning or replacing a default voice must persist with `Save Character`; do not silently save character edits unless the existing editor already uses auto-save.
- The Gallery card surfaces default voice state with a compact badge:
  - Assigned: `Voice: {voice name}` or truncated voice name with accessible full text.
  - Missing: `No voice`.
  - Broken reference: `Voice unavailable` with warning icon.
- Gallery voice badges must not crowd portrait, card actions, or text snippets. Keep card dimensions stable.

### Settings

- Preserve the Phase 01.1 save-before-test rule: pressing any `Test Connection` button first persists current form values, then tests the endpoint.
- Endpoint sections required in Phase 2: Web UI, AI backend, and LLM. Each section shows `Connected`, `Unreachable`, `Unauthorized`, or `Not configured`.
- AI backend status includes model residency summary when available: STT model, VAD readiness, resident TTS engine, and VRAM/headroom. Keep this as compact status text, not a dashboard.
- Add save-audio defaults:
  - `Save AI audio` default ON.
  - `Save mic audio` default OFF.
- Save-audio toggles use switch controls with explicit helper text. Mic audio helper must state privacy implication: `Off by default; future calls will not store your microphone audio unless enabled.`
- Add VAD sensitivity placeholders without pretending Phase 4 wiring is complete. Controls may be disabled or marked `Coming in Call Feel` if backend wiring is not ready.
- VAD placeholders include threshold and end-of-utterance silence duration labels. Use sliders only if values can be stored; otherwise use read-only rows.
- STT model and TTS default engine controls may appear only if backed by Settings payloads. If present, label STT default as `distil-large-v3` and TTS default as `F5-TTS`.
- Do not add clear-all-data danger zone in Phase 2 unless backend support already exists; full danger zone belongs to later Settings scope.

## Interaction And State Contracts

### Loading

- Use tonal skeletons for Voice Library rows and Settings panels.
- Use inline progress for upload, transcription, synthesis, save, rename, delete, and connection tests.
- Transcription and synthesis loading states must be scoped to the active sample/voice and must not blank the entire page.
- Audio controls reserve fixed dimensions before audio is ready.

### Empty

- Empty Voice Library uses `No voices yet` and the body from the Copywriting Contract.
- Empty default voice assignment uses `No voice assigned`.
- Empty transcript before upload is not an error; show upload guidance and keep transcript editor disabled or hidden until a sample exists.

### Error

- Upload errors distinguish unsupported type, unreadable file, duration detection failure, and backend upload failure.
- Transcript errors keep the uploaded sample visible and provide retry.
- Preview errors keep transcript edits and engine selection intact.
- Save errors keep all form state intact and explain whether the failure was validation, storage, or backend synthesis.
- Delete blocked errors list referents by readable character/chat name. Do not show raw IDs as the primary label.
- Settings connection errors identify the endpoint that failed and keep the existing status vocabulary.

### Focus And Keyboard

- Dropzone is keyboard reachable and triggers file selection with Enter or Space.
- Engine picker supports arrow-key or tab navigation with visible selected and focused states.
- Audio play/pause, retry, rename, delete, and reassignment controls have accessible labels.
- Dialogs trap focus and close on Escape unless a destructive operation is submitting.
- Disabled preview/save controls expose why they are disabled through nearby helper text, not only `title` attributes.

### Motion

- Acceptable motion: upload drag transition, transcript pending shimmer, preview progress pulse, compact audio level bars, status-chip transition, and modal/menu fade.
- Respect `prefers-reduced-motion` by disabling shimmer, pulse, animated gradients, and audio bar animation.
- No decorative orbs, bokeh blobs, or marketing animation.

### Audio Playback

- Only one preview/test audio clip plays at a time. Starting a new preview or test-play pauses any currently playing Voice Lab/Library clip.
- Audio errors surface inline with retry and do not silently fail.
- Preview/test controls must work without assuming a call audio device picker exists in Phase 2.

## Responsive Contract

- Desktop: app shell rail plus two-column Voice Lab workspace.
- Tablet: rail may collapse; Voice Lab remains two-column only if both columns keep at least 320px.
- Mobile: single-column Voice Lab; creation flow appears before Voice Library; sticky preview/save action bar is allowed but must not overlap bottom nav.
- Upload controls, engine picker, transcript editor, and audio preview controls must fit within a 320px viewport without horizontal scroll.
- Long voice names, engine labels, endpoint URLs, model names, and referent lists truncate or wrap with accessible full text.
- Gallery voice badges must remain legible on mobile cards and must not overlap overflow menus.

---

## Accessibility And Safety

- Color is never the only state indicator. Pair every connected/error/warning/selected/default/blocked state with text and icon or shape.
- All audio controls need accessible names and visible focus.
- Use native `audio` where practical for browser accessibility; custom controls must preserve play/pause semantics.
- Confirm destructive actions before execution. The confirmation action button repeats the destructive verb.
- Deleting a voice must never leave dangling default voice or chat override references. UI must either guide reassignment or block with referents.
- API keys remain masked and must not appear in status copy, toast copy, screenshots, or console output.
- Voice sample file names and transcripts are user-provided content; render as text only.
