<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser media APIs -> RayMe call route | Microphone permission and media streams enter the app from the browser and user device. |
| Client route -> Web UI `/api/calls` | Same-origin typed wrappers send call controls and reconnect backfill. |
| Browser diagnostics -> Web UI debug route | Client runtime details can cross to the server for debugging. |
| Mobile viewport -> call controls | Layout determines whether touch controls remain usable on Android Chrome. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03.1-05-01 | Spoofing | microphone permission/startup | mitigate | Start call only through browser `getUserMedia()` in HTTPS secure context; denial shows fixed retry copy and does not create a live backend session. |
| T-03.1-05-02 | Tampering | same-origin call wrappers | mitigate | Route consumes Plan 03.1-04 `/calls/...` wrappers and does not introduce direct AI backend/provider URLs. |
| T-03.1-05-03 | Denial of Service | reconnect backfill | mitigate | Keep `MIC_BACKFILL_MAX_MS = 30000`, `MIC_BACKFILL_BATCH_MAX_MS = 10000`, and bounded `MEDIA_RECONNECT_MAX_ATTEMPTS = 2`. |
| T-03.1-05-04 | Repudiation | terminal media failure | mitigate | Terminal failure performs recover-before-end cleanup and shows truthful failed/ended copy instead of optimistic continuation. |
| T-03.1-05-05 | Information Disclosure | diagnostics | mitigate | Use existing bounded debug event route; no raw secrets, tracebacks, or unbounded log spam are added to committed evidence. |
| T-03.1-05-06 | Denial of Service | mobile controls | mitigate | Playwright mobile asserts 44px touch targets and toolbar position above AppShell bottom navigation. |
</threat_model>
