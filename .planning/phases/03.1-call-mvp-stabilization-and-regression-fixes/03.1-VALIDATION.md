## Sampling Rate

- **After every task commit:** run the quick command for the tier touched by that task.
- **After every cross-tier boundary task:** run the relevant backend/server/client quick command plus one browser call smoke.
- **After every plan wave:** run the focused call baseline across AI backend, Web UI server, client unit, desktop Playwright, and mobile Playwright if client media/layout changed.
- **Before `$gsd-verify-work`:** deploy only through `scripts/deploy-omen.sh`, then run non-mocked live desktop call evidence at `https://192.168.1.199:8443`.
- **Android acceptance:** required after agent-run local and OMEN desktop verification when changes touch mobile Chrome, mic permission, call startup, audio playback, or layout.
- **Max feedback latency:** no more than one task commit without an automated tier check; no more than one wave without the focused call baseline.

## Security Threat References

| Ref | Threat | Required Mitigation |
|-----|--------|---------------------|
| T-01 | Cross-origin call control POSTs | Preserve `enforce_same_origin_for_calls` on browser-facing call routes |
| T-02 | Oversized reconnect/audio payloads | Keep Pydantic length/range bounds and backend audio size checks |
| T-03 | Optimistic call continuation after media failure | Prefer truthful failed/ended states after bounded recovery fails |
| T-04 | Diagnostic secret leakage | Do not commit API keys, tokens, passwords, raw traces, absolute local paths, or model-cache internals |
| T-05 | CPU fallback or runtime mismatch on OMEN | Use `scripts/deploy-omen.sh` CUDA/runtime guards only |
| T-06 | Browser mic on insecure context | Require LAN HTTPS secure context and Android trusted certificate workflow |
