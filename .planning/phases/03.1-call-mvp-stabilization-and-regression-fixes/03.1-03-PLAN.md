<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> Web UI `/api/calls` | Untrusted browser call control requests cross into the server facade. |
| Web UI server -> AI backend | Server forwards owned `call_id`/`session_id` pairs and bounded payloads to transient backend sessions. |
| Diagnostics -> committed/logged evidence | Browser/runtime details may contain sensitive strings if not bounded and sanitized. |
| Durable DB -> prompt builder | Stored thread rows become LLM prompt context. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03.1-03-01 | Tampering | `/api/calls` unsafe POST routes | mitigate | Preserve `enforce_same_origin_for_calls` on start, offer, mute, interrupt, turns, reconnect-audio, events/recover, end, and debug event routes. |
| T-03.1-03-02 | Denial of Service | reconnect audio facade | mitigate | Preserve Pydantic bounds matching the AI backend: `pcm_b64 max_length=4_000_000`, sample rate/channel bounds, and no persistence from reconnect payloads. |
| T-03.1-03-03 | Spoofing | call/session ownership | mitigate | Reject posted `session_id` mismatches before proxying controls or writeback. |
| T-03.1-03-04 | Information Disclosure | `/_debug/event` and backend errors | mitigate | Truncate detail to 800 characters, log only, no DB write, no backend call, and keep public backend error mapping. |
| T-03.1-03-05 | Repudiation | durable call end | mitigate | Persist `call_end` rows for non-misconfiguration cleanup failures so transcript state remains truthful. |
| T-03.1-03-06 | Tampering | call prompt context | mitigate | Build prompt context with selected non-stale text/speech rows and exclude call boundary events. |
</threat_model>
