<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser/Web UI server -> AI backend `/webrtc` | Untrusted call control, offer, speak, and reconnect payloads enter the AI backend. |
| Browser media -> `CallSession` | Live audio frames can be large, malformed, delayed, duplicated, or arrive during muted/terminal states. |
| AI backend runtime -> public API response | Model and runtime exceptions cross back to browser-visible code. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03.1-02-01 | Denial of Service | reconnect audio payloads | mitigate | Preserve Pydantic bounds `pcm_b64 max_length=4_000_000`, `sample_rate 8000..48000`, `channels 1..2`, `attempt 0..10`, and `batch_index 0..20`. |
| T-03.1-02-02 | Information Disclosure | `/webrtc/speak` failures | mitigate | Keep fixed public `call_tts_failed` details and add tests rejecting traceback, local path, and model-cache leaks for `voxcpm2` and `f5`. |
| T-03.1-02-03 | Repudiation | reconnect/drop handling | mitigate | Failed/closed media paths set `end_reason == "connection_failed"` so evidence can distinguish terminal failure from normal end. |
| T-03.1-02-04 | Tampering | mute control | mitigate | Backend mute state drops inbound frames and counts `dropped_audio_frames`; browser local track state alone cannot satisfy tests. |
| T-03.1-02-05 | Denial of Service | duplicate reconnect backfill | mitigate | Deduplicate `backfill_id` values and return `status: "duplicate"` without re-appending PCM. |
| T-03.1-02-06 | Tampering | VoxCPM2 playback branch | mitigate | Preserve Phase 8 no-fallback streaming branch and verify `streaming_used=true` plus `whole_wav_fallback_used=false` for VoxCPM2 metrics. |
</threat_model>
