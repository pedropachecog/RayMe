## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Recovered aggregate connection state still timed out to connection_failed**
- **Found during:** Task 1 (Add backend RED regressions for call lifecycle invariants)
- **Issue:** `_receive_audio_track()` entered the disconnected retry path when `connectionState == "disconnected"`, but the retry success condition only checked ICE state. A peer whose aggregate `connectionState` recovered to `connected` could still be failed after bounded retries.
- **Fix:** Updated the retry loop to treat either `iceConnectionState in {"completed", "connected"}` or `connectionState == "connected"` as recovered, and to return without failing if the session ended during retry cleanup.
- **Files modified:** `ai-backend/app/api/webrtc.py`, `ai-backend/tests/test_webrtc_signaling.py`
- **Verification:** `uv run --project ai-backend pytest ai-backend/tests/test_call_session.py ai-backend/tests/test_webrtc_signaling.py -q` passed with 71 tests.
- **Committed in:** `ab7eb82`
