<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| card upload -> parser | Untrusted JSON/PNG bytes enter backend parsing. |
| portrait upload -> filesystem | Untrusted image bytes become local files. |
| character delete -> historical threads | User deletion affects persisted conversation references. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-12-XSS | E/T | `cards.py`, character API | mitigate | Preserve untrusted HTML as data; client sanitizer renders it later. |
| T-01-12-PNGDOS | D | `cards.py` | mitigate | Enforce upload/metadata limits with safe errors. |
| T-01-12-TRAVERSAL | T/I | `portraits.py` | mitigate | Reject path-like filenames and use server-generated blob IDs. |
| T-01-12-DATADELETE | D | `character_service.py` | mitigate | Soft delete characters and preserve thread snapshots/messages. |
</threat_model>
