<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Alembic migration -> SQLite file | Schema changes create irreversible durable state. |
| API/domain -> filesystem blobs | Untrusted uploads become local files. |
| character delete/message branch -> history | User actions can remove or corrupt conversation history. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04-SCHEMA | T/D | `0001_initial_schema.py` | mitigate | Migration tests require all message kinds, alternates, stale flags, and snapshots before APIs depend on them. |
| T-01-04-TRAVERSAL | T/I | `blob_store.py` | mitigate | Reject path separators/absolute paths and use server-generated blob names only. |
| T-01-04-ORPHAN | D | `reaper.py` | mitigate | Reaper limited to configured blob directory and tested against outside deletion. |
| T-01-04-DELETE | D | `characters`, `threads` tables | mitigate | Soft delete and historical thread snapshot fields prevent character delete from orphaning chat history. |
| T-01-04-BRANCHDATA | T/D | `messages`, `message_alternates` tables | mitigate | Durable branch/stale columns prevent regenerate/swipe/edit/continue data loss. |
</threat_model>
