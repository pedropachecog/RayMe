<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LAN browser -> FastAPI app | Unauthenticated local clients call API and static assets. |
| FastAPI app -> AI backend | Web host probes a separate LAN service. |
| FastAPI app -> OpenAI-compatible LLM | Server uses sensitive endpoint/key/model values. |
| HTTPS config -> network interface | Runtime bind and TLS settings determine LAN exposure. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05-BIND | I/T | `config.py`, `run_dev_https.py` | mitigate | Reject `0.0.0.0` and require explicit host/IP; tests cover rejection. |
| T-01-05-CORS | S/T | `security.py`, `main.py` | mitigate | Reject wildcard origins and configure explicit CORS allowlist. |
| T-01-05-KEY | I | `settings.py`, `llm_probe.py` | mitigate | Store/probe API key server-side; return only `llm_api_key_configured` and status strings. |
| T-01-05-XSS | E | CSP middleware | mitigate | Add CSP headers limiting default/connect sources while sanitizer handles content. |
| T-01-05-LLMHEALTH | S/T | `llm_probe.py` | mitigate | Implement LLM health as a server-side configured endpoint probe, not a fake local LLM service. |
</threat_model>
