<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| card upload -> parser | Untrusted JSON/PNG bytes enter backend parsing. |
| portrait upload -> filesystem | Untrusted image bytes become local files. |
| character delete -> historical threads | User deletion affects persisted conversation references. |
| lorebook storage -> prompt builder | Preserved world-info could contaminate LLM prompts if wired incorrectly. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-06-XSS | E/T | `cards.py`, character API | mitigate | Preserve untrusted HTML as data only; sanitizer is the only render path in client plans. |
| T-01-06-PNGDOS | D | `cards.py` | mitigate | Enforce 10 MiB upload and 1 MiB decoded card JSON limits with safe errors. |
| T-01-06-TRAVERSAL | T/I | `portraits.py` | mitigate | Reject path-like filenames and use server-generated blob IDs plus atomic writes. |
| T-01-06-PROMPT | T | `lorebook_json` | mitigate | Store lorebook/world-info and emit present-not-used warning; prompt builder tests in Plan 07 exclude it. |
| T-01-06-DATADELETE | D | `character_service.py` | mitigate | Soft delete characters and preserve thread snapshots/messages. |
</threat_model>
