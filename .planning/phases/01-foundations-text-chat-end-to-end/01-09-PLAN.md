<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| imported card text -> Gallery/Editor DOM | Untrusted card fields render in browser. |
| user upload -> character API | JSON/PNG/portrait files cross from browser to backend. |
| Settings API key input -> browser UI | Sensitive LLM key is typed into the UI. |
| delete actions -> persistent history | Character/thread deletion can remove data. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-09-XSS | E/T | `CharacterCard.svelte`, Editor previews | mitigate | Route all snippets/prose through `renderTrustedMarkdown`; tests use malicious payload. |
| T-01-09-UPLOAD | T/D | `ImportCardDialog`, `PortraitDropzone` | mitigate | Restrict accepted extensions and surface backend unsafe/malformed errors explicitly. |
| T-01-09-KEY | I | `EndpointSettingsPanel.svelte` | mitigate | Mask API key by default, reveal by icon only, and never render key in status/toasts. |
| T-01-09-DELETE | D | `ConfirmDialog` | mitigate | Destructive confirmations use exact copy warning that character delete preserves existing chats while thread delete removes conversation history. |
| T-01-09-FUTURE | T | Settings/Editor/Gallery | mitigate | Grep/tests block out-of-scope Voice/Call/billing/PWA/VAD controls. |
</threat_model>
