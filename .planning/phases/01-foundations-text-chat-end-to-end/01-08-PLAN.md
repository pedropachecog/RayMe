<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client API wrapper -> backend | Browser code can choose which URL to call. |
| backend stream -> chat UI | Token stream drives user-visible assistant output. |
| browser capability checks -> UI | Secure-context status affects mobile readiness messaging. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-08-KEY | I | `api/client.ts`, `settings.ts` | mitigate | Reject absolute external URLs and never store API key values in client code. |
| T-01-08-STREAM | T/D | `stream.ts` | mitigate | Parse typed SSE JSON events and expose explicit error handling. |
| T-01-08-XSS | E | Shell/status components | transfer | Sanitizer implemented in Plan 02; feature screens must use it for untrusted card text. |
| T-01-08-UXSCOPE | T | `AppShell.svelte` | mitigate | Tests and grep prevent fake future controls that could imply unavailable auth/call/voice behavior. |
| T-01-08-MOBILE | D | `environment.ts` | mitigate | Text-based secure-context/media readiness status surfaces mobile blockers early. |
</threat_model>
