<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> chat API | User text enters storage and LLM prompt context. |
| server -> LLM endpoint | Server-side secret settings are used to stream completions. |
| message actions -> SQLite | Edits/regenerates/swipes mutate durable branch state. |
| character lorebook -> prompt builder | Preserved lorebook data must not become prompt input. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-07-KEY | I | `chat.py`, `llm_stream.py` | mitigate | Source base URL/key/model from server settings only; tests reject browser override and key echo. |
| T-01-07-PROMPT | T | `prompt_builder.py` | mitigate | Tests exclude lorebook/world-info and unselected alternates from context. |
| T-01-07-BRANCHDATA | T/D | `message_actions.py` | mitigate | Tests cover selected alternate, stale flags, truncate/keep, and regenerate replacement. |
| T-01-07-STREAM | D/T | `llm_stream.py` | mitigate | Persist final AI message once after completion; error event does not store partial final text. |
| T-01-07-DELETE | D | `threads.py` | mitigate | Thread delete is explicit to thread only; character delete preservation handled by Plan 06. |
</threat_model>
