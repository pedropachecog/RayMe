<objective>
Repair the invalid `1806eb0` live-call TTS fix and install durable guardrails so future agents cannot trade RayMe's phone-call behavior for full-response buffering.

Output: VoxCPM2 live calls use only bounded startup/jitter buffering before first playback, never full-stream buffering; tests and planning docs explicitly reject generate-then-play call fixes; corrected code is committed, pushed, and deployed to OMEN through `scripts/deploy-omen.sh`.
</objective>

<user_goal_preservation>
The user must still be able to have a live RayMe phone call where the assistant response appears and plays in the call, playback begins before full TTS generation completes, the call does not sit indefinitely in Rehearsing, and the call remains interruptible/listenable afterward.
</user_goal_preservation>
