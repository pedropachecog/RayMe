# CHECKPOINT: Explain And Resolve Invalid Discussion Fallback

## Why This Exists

`$gsd-discuss-phase 3` must not proceed as if Phase 3 context already exists or
as if inferred defaults are user decisions. A previous run wrote a draft context
without asking the user any questions.

Before continuing Phase 3 discussion, the agent must demonstrate the blocking
anti-pattern below, then ask the user Phase 3 questions in plain text and wait
for answers.

## Critical Anti-Patterns

| severity | anti-pattern | manifestation | structural guard |
|---|---|---|---|
| blocking | Treating unavailable prompt UI as permission to infer decisions | Phase 3 context was written without asking the user anything | Use plain-text numbered choices and stop for user answers; canonical context requires actual user answers or explicit `--auto`/`--chain` |
