## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, waveform bar width, compact inline chips |
| sm | 8px | Compact element spacing, transcript row gaps, toolbar icon-label gaps |
| md | 16px | Default panel padding, call transcript bubble gaps |
| lg | 24px | Call screen region padding, modal padding |
| xl | 32px | Desktop workspace gutters, visualizer-to-transcript spacing |
| 2xl | 48px | Major call layout separation on desktop |
| 3xl | 64px | Page-level spacing and mobile bottom-nav reserve when present |

Exceptions: call controls must have 44px minimum touch targets; active call primary controls should be 56px square; mobile call control bar reserves at least 80px plus `env(safe-area-inset-bottom)`; transcript bottom padding must be at least 96px when controls are sticky.

## Typography

Use exactly these four sizes and two weights for Phase 3 UI: 400 regular and 600 semibold.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 | 1.5 |
| Label | 12px | 600 | 1.3 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.15 |

Call transcript body text uses Body. Toolbar labels, status chips, device picker helper text, and transcript state chips use Label. Call screen title and blocking-state headings use Heading. Display is reserved for the central call status or active visualizer label only, not for ordinary panels.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#060e20` | Page background, call canvas, inactive transcript space |
| Secondary (30%) | `#141f38` | Glass panels, transcript bubbles, toolbar surface; use `#091328` for rail/workspace backing |
| Accent (10%) | `#b6a0ff` / pulse gradient `linear-gradient(135deg, #b6a0ff 0%, #70aaff 100%)` | Start Call CTA, active stream caret, interrupt affordance, selected device state, focus glow |
| Destructive | `#ff716c` | End Call, failed call state, destructive confirmation only |

Accent reserved for: Start Call, active AI streaming indicator, button-based Interrupt, selected/active call controls, focus-visible outline, and the primary Voice Visualizer glow. Do not use accent for every clickable element.

Communication color: `#00e3fd` is reserved for live listening/speaking status, healthy connection dots, and RMS-driven visualizer energy. It is not a general CTA color.

No-line rule: do not add 1px section borders. Separate call regions with tonal surfaces, spacing, blur, and ghost outlines only when a control boundary needs accessibility contrast.

## Phase Interaction Contract

### Entry And Preflight

- Calls start from the chat thread header and from character cards.
- Character-card start must create or select the thread before entering the call surface.
- Start Call runs preflight before showing the active call screen: secure context, mic permission, usable assigned voice, AI backend readiness, and required model readiness.
- Failed preflight stays out of the active call screen and shows one blocking recovery panel using the copy above.

### Call Screen Layout

- Use a minimal operational call screen, not a final-product call-feel layout.
- Desktop layout: top status row, central Voice Visualizer, live transcript column, sticky bottom toolbar.
- Mobile layout: single-column call canvas with visualizer above transcript and sticky controls at the bottom. The AppShell bottom nav must be hidden or fully reserved so it never overlaps Mute, Interrupt, End Call, or device controls.
- Keep typed composer hidden while a call is active. Text entry resumes only after hangup on the originating thread.

### Live State Model

Use these user-visible states:

| State | Visual Contract |
|-------|-----------------|
| `connecting` | Muted glass surface, status chip `Connecting`, controls disabled except End Call |
| `listening` | Cyan RMS pulse, status chip `Listening`, Mute and End Call enabled |
| `thinking` | Primary gradient shimmer, status chip `Thinking`, Interrupt and End Call enabled |
| `speaking` | Cyan/blue waveform driven by AI output, status chip `Speaking`, Interrupt and End Call enabled |
| `interrupted` | Short `Interrupted` chip, then return to `Listening`; do not keep a persistent error panel |
| `ended` | Summary panel with duration and `Return to Thread` |
| `failed` | Destructive-tone panel using the peer/network failure copy |

### Transcript

- The live transcript is visible during the call.
- User turns appear only after finalization.
- AI text streams forward-stable once generation starts; visible text must not be rewritten.
- Streaming AI turns use the existing gradient caret treatment.
- Transcript rows use existing bubble proportions and 14px body text; call rows may add compact Label chips such as `Final`, `Streaming`, or `Interrupted`.
- After End Call, the originating thread scrollback must show `call_start`, `user_speech`, `ai_speech`, and `call_end` rows in chronological order.

### Toolbar

- Toolbar controls: Mute, Interrupt during AI turns, audio input picker, audio output picker, End Call.
- Mute must visibly toggle between `Mute` and `Unmute` and reflect server-side consumption state, not local silence only.
- Unsupported device pickers remain visible, disabled, and expose the copy above through adjacent helper text or accessible description.
- End Call uses destructive color and a `PhoneOff` icon. Interrupt uses an explicit label because it is a new call-specific action.

### Voice Visualizer

- Phase 3 must include the three core visualizer states: listening, thinking, speaking.
- MVP fidelity is acceptable: CSS bars/pulses driven by RMS where available, deterministic animated fallback when metering is unavailable.
- Respect `prefers-reduced-motion: reduce` by switching to static state color and status chips.
