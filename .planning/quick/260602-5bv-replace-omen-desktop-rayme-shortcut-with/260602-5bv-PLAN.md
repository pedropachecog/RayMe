# Quick Plan: Replace OMEN Desktop RayMe Shortcut With Debuggable Console

## Goal

Make the OMEN desktop launcher for RayMe feel like a local debuggable app:

- one visible console window,
- AI and Web logs streamed in that console,
- URL printed clearly,
- no automatic browser opening,
- closing the console stops RayMe.

Also preserve the AIbert communication/profile contract in workspace-readable
files so future sessions do not repeat the same interaction failure.

User-goal preservation: this fix must make the Desktop launcher visibly run
RayMe in one debuggable console with logs and an obvious stop path; it must not
hide runtime state behind scheduled tasks, background windows, auto-opened
browser tabs, or quiet success messages.

## Scope

- Update `AGENTS.md` and `.planning/` profile/startup notes with the AIbert
  contract and RayMe UX preference.
- Replace `scripts/start-rayme-omen.ps1` with a foreground process manager.
- Update `scripts/deploy-omen.sh` so the Desktop shortcut launches that console
  normally.
- Keep scheduled task deployment canonical through `scripts/deploy-omen.sh`.
- Update docs and GSD state artifacts.

## Verification

- PowerShell syntax check for `scripts/start-rayme-omen.ps1`.
- Shell syntax check for `scripts/deploy-omen.sh`.
- Deploy to OMEN via `scripts/deploy-omen.sh`.
- Confirm Desktop shortcut target/arguments.
- Confirm no browser auto-opens from the launcher.
- Confirm the launcher prints logs and stop guidance.
